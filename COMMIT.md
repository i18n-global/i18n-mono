# 커밋 메시지 작성 원칙

## 기본 규칙

1. **최대 2줄 이내** - 첫 번째 줄: 요약, 두 번째 줄: 선택적 상세 설명
2. **동사형 한국어** - 명사형이 아닌 동사형으로 작성 (예: "구현", "수정", "추가")
3. **짧고 간결하게** - 함수 이름 나열하지 말고 기능 중심으로 설명

## 형식

```
<type>: <동사형 설명>
<선택적 상세 설명>
```

## Type 접두사

- `feat:` - 새로운 기능 추가
- `fix:` - 버그 수정
- `test:` - 테스트 추가/수정
- `refactor:` - 코드 리팩토링
- `chore:` - 빌드/설정 변경
- `docs:` - 문서 수정

## 예시

```bash
feat: Sentry 제거 및 콘솔 로깅만 사용
chore: 불필요한 파일 정리 및 커밋 가이드 추가
```
