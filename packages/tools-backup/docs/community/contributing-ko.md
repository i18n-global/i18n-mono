# 기여 가이드

i18nexus-tools에 기여해주셔서 감사합니다! 이 가이드는 프로젝트에 기여하는 방법을 안내합니다.

## 🤝 기여 방법

### 기여 유형

다양한 유형의 기여를 환영합니다:

- **버그 리포트**: 발견한 문제를 보고
- **기능 요청**: 새로운 기능이나 개선사항 제안
- **코드 기여**: 버그 수정이나 새로운 기능 제출
- **문서화**: 문서 개선이나 번역
- **테스트**: 새로운 기능 테스트 및 문제 보고

### 시작하기

1. **저장소 포크**
2. **포크한 저장소 클론**
3. **기능 브랜치 생성**
4. **변경사항 작성**
5. **변경사항 테스트**
6. **풀 리퀘스트 제출**

## 🚀 개발 환경 설정

### 사전 요구사항

- Node.js >= 18.0.0
- npm >= 9.0.0
- Git

### 설정 단계

```bash
# 저장소 포크 및 클론
git clone https://github.com/YOUR_USERNAME/i18nexus.git
cd i18nexus/packages/i18nexus-tools

# 의존성 설치
npm install

# 프로젝트 빌드
npm run build

# 테스트 실행
npm test
```

### 개발 명령어

```bash
# 개발용 워치 모드
npm run dev

# TypeScript 빌드
npm run build

# 테스트 실행
npm test

# 코드 린팅
npm run lint

# 코드 포맷팅
npm run format
```

## 📝 코드 스타일

### TypeScript 가이드라인

- 모든 새 코드에 TypeScript 사용
- 기존 코드 패턴 따르기
- 의미있는 변수 및 함수명 사용
- 공개 API에 JSDoc 주석 추가

### 코드 포맷팅

Prettier를 사용한 코드 포맷팅:

```bash
# 모든 코드 포맷팅
npm run format

# 포맷팅 확인
npm run format:check
```

### 린팅

ESLint를 사용한 코드 린팅:

```bash
# 모든 코드 린팅
npm run lint

# 린팅 문제 수정
npm run lint:fix
```

## 🧪 테스트

```bash

npm run test:watch
```

## 🔄 풀 리퀘스트 프로세스

### 제출 전 확인사항

1. **기존 이슈 확인** - 이미 보고된 문제가 아닌지 확인
2. **이슈 생성** - 중요한 변경사항의 경우 먼저 이슈 생성
3. **포크 및 브랜치** - main에서 기능 브랜치 생성
4. **변경사항 작성** - 변경사항 구현
5. **충분한 테스트** - 모든 테스트 통과 확인
6. **문서 업데이트** - 관련 문서 업데이트

### 풀 리퀘스트 가이드라인

- **명확한 제목** - PR이 무엇을 하는지 설명
- **상세한 설명** - 변경사항과 이유 설명
- **이슈 연결** - 관련 이슈 참조
- **작은 변경사항** - PR을 집중적이고 관리 가능하게 유지
- **테스트 커버리지** - 새로운 기능에 대한 테스트 포함

### PR 템플릿

```markdown
## 설명

변경사항에 대한 간단한 설명

## 변경 유형

- [ ] 버그 수정
- [ ] 새로운 기능
- [ ] 호환성 깨지는 변경
- [ ] 문서 업데이트

## 테스트

- [ ] 테스트 통과
- [ ] 수동 테스트 완료
- [ ] 새로운 테스트 추가

## 체크리스트

- [ ] 코드가 스타일 가이드라인을 따름
- [ ] 자기 검토 완료
- [ ] 문서 업데이트됨
- [ ] 호환성 깨지는 변경 없음 (또는 문서화됨)
```
